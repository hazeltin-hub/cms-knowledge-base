# CMS 报表与分析指南

## 标签
#user-guide #report #analytics #advanced

## 概述

本文档介绍如何使用 Storellet CMS 的报表和分析功能，包括数据概览、各类报表查看、数据导出和自定义报表。

## 📊 访问报表系统

### 步骤 1：进入报表页面

1. 登录 CMS 系统
2. 点击顶部导航的**"报表"**或**"Report"**
3. 系统会跳转到报表首页
   ```
   URL: /cms/report/index
   ```

### 步骤 2：选择品牌和日期范围

在报表页面：
- **品牌**：选择要查看的品牌
- **开始日期**：选择报表起始日期
- **结束日期**：选择报表结束日期

## 📈 报表概览（Overview）

### 访问概览页面

1. 点击**"概览"**或**"Overview"**
2. 系统会显示关键指标的汇总数据
   ```
   URL: /cms/report/overview
   ```

### 概览显示内容

#### 1. 折扣卡数据（Discount Cards）

显示所有折扣卡的统计信息：

| 指标 | 说明 |
|------|------|
| **期间领取数** | 选择时间段内领取折扣卡的用户数 |
| **总领取数** | 从开始到现在的总领取数 |
| **领取率** | 领取用户数 / 总用户数 |

**查看方式**：
- 显示所有活跃的折扣卡
- 每个折扣卡显示期间领取数和总领取数
- 可以比较不同折扣卡的表现

#### 2. 欢迎礼包优惠券（Welcome Pack Coupons）

新用户注册时自动获得的优惠券：

| 指标 | 说明 |
|------|------|
| **期间发放数** | 选择时间段内发放的优惠券数量 |
| **总发放数** | 从开始到现在的总发放数 |
| **期间使用数** | 选择时间段内使用的优惠券数量 |
| **总使用数** | 从开始到现在的总使用数 |
| **期间过期数** | 选择时间段内过期的优惠券数量 |
| **总过期数** | 从开始到现在的总过期数 |

**计算公式**：
- **使用率** = 使用数 / 发放数
- **过期率** = 过期数 / 发放数

**用途**：
- 评估欢迎礼包的吸引力
- 分析新用户的首次使用行为
- 优化欢迎礼包内容

#### 3. 印花优惠券（Stamp Coupons）

通过收集印花获得的优惠券：

| 指标 | 说明 |
|------|------|
| **期间发放数** | 选择时间段内完成印花卡并发放的优惠券 |
| **总发放数** | 总发放的印花优惠券数量 |
| **期间使用数** | 选择时间段内使用的印花优惠券 |
| **总使用数** | 总使用的印花优惠券数 |
| **期间过期数** | 选择时间段内过期的印花优惠券 |
| **总过期数** | 总过期的印花优惠券数 |

**分析要点**：
- 印花卡完成率
- 印花优惠券的使用率
- 优惠券过期率

#### 4. 推送优惠券（Push Coupons）

通过推送消息发放的优惠券：

| 指标 | 说明 |
|------|------|
| **期间发放数** | 选择时间段内通过推送发放的优惠券 |
| **总发放数** | 总推送发放的优惠券数 |
| **期间使用数** | 选择时间段内使用的推送优惠券 |
| **总使用数** | 总使用的推送优惠券数 |
| **期间过期数** | 选择时间段内过期的推送优惠券 |
| **总过期数** | 总过期的推送优惠券数 |

**分析要点**：
- 推送效果分析
- 推送优惠券的吸引力
- 推送时间和内容优化

### 概览最佳实践

#### 数据对比分析

**时间对比**：
- 选择不同时间段对比数据
- 例如：本月 vs 上月
- 例如：活动期间 vs 平时

**品牌对比**：
- 切换不同品牌查看数据
- 对比同一时期不同品牌的表现

**指标分析**：
- 关注使用率（使用数 / 发放数）
- 关注过期率（过期数 / 发放数）
- 使用率过低：可能是优惠券不够吸引
- 过敏率过高：可能是有效期太短

## 📥 导出报表

### 导出步骤

1. 在报表页面
2. 设置好筛选条件
3. 点击**"导出"**或**"Export"**按钮
   ```
   URL: /cms/report/{type}/{reqBrandId}/export
   ```
4. 选择导出格式：
   - **Excel** (.xlsx) - 推荐
   - **CSV** - 用于数据分析

### 导出内容

导出的数据包含：

**基本信息**：
- 报表名称
- 日期范围
- 品牌信息
- 导出时间

**数据表格**：
- 根据报表类型包含不同字段
- 包含所有筛选后的数据
- 数据格式化处理

**汇总统计**：
- 总数统计
- 百分比计算
- 趋势分析

### 导出最佳实践

#### 文件命名

建议的命名格式：
```
[报表名称]_[品牌]_[开始日期]_[结束日期].xlsx
例如：
用户报表_BrandA_20260101_20260131.xlsx
优惠券使用统计_BrandB_202601.xlsx
```

#### 数据分析

**Excel 分析**：
- 使用数据透视表分析
- 创建图表可视化
- 计算衍生指标

**定期导出**：
- 每月导出关键报表
- 保存历史数据对比
- 建立数据趋势分析

## 🔔 Coown 报警管理

### 什么是 Coown 报警

Coown 报警是一种监控和通知机制，用于在特定事件或条件触发时发送通知。

### 查看报警列表

1. 点击**"Coown 报警"**或**"Coown Alarm"**
2. 系统会显示所有报警配置
   ```
   URL: /cms/report/coownAlarm
   ```

### 报警信息

每个报警配置包含：
- **报警名称**：报警的标识
- **执行时间**：报警触发的默认时间
- **状态**：启用/禁用
- **配置详情**：报警的具体配置

### 创建报警

1. 点击**"新建报警"**
2. 填写报警信息：
   - 报警名称
   - 执行时间
   - 报警条件
   - 通知方式
3. 点击**"保存"**
   ```
   URL: /cms/report/coownAlarm/{id}
   ```

### 编辑报警

1. 在报警列表中找到要编辑的报警
2. 点击报警名称进入详情页面
   ```
   URL: /cms/report/coownAlarm/{id}
   ```
3. 修改报警配置
4. 点击**"保存"**

### 删除报警

1. 在报警列表中找到要删除的报警
2. 点击**"删除"**
   ```
   URL: /cms/report/coownAlarm/{id}/delete
   ```
3. 确认删除

### 报警执行时间

默认执行时间（DEFAULT_EXECUTION_TIME）：
- 系统会在设定的时间检查报警条件
- 如果条件满足，发送报警通知
- 可以自定义每个报警的执行时间

2. 系统会显示该类型的所有报表列表
   ```
   URL: /cms/report/storelletReports/{reportType}
   ```
3. 点击报表名称查看详情

### 创建自定义报表

#### 步骤 1：进入创建页面

1. 点击**"新建报表"**
2. 系统会显示报表创建页面
   ```
   URL: /cms/report/storelletReportView
   ```

#### 步骤 2：配置报表

填写以下信息：

**基本信息**：
- **报表名称**：报表的显示名称
- **报表描述**：报表的说明
- **报表类型**：选择报表类型（1-5）

**数据源配置**：
- **时间范围**：默认查询的时间范围
- **品牌筛选**：是否包含品牌筛选
- **店铺筛选**：是否包含店铺筛选

**显示配置**：
- **图表类型**：选择图表显示方式
  - 折线图 - 趋势分析
  - 柱状图 - 对比分析
  - 饼图 - 占比分析
  - 表格 - 详细数据

**指标配置**：
- 选择要显示的指标
- 设置指标的计算方式
- 配置指标的显示格式

#### 步骤 3：保存报表

1. 点击**"保存"**
   ```
   URL: /cms/report/storelletReport (POST)
   ```
2. 系统会验证配置
3. 保存成功后可以在报表列表中找到

### 查看报表数据

1. 在报表列表中点击报表名称
   ```
   URL: /cms/report/storelletReportView/{id}
   ```
2. 系统会显示：
   - 报表配置信息
   - 查询条件筛选
   - 数据可视化图表
   - 详细数据表格

### 修改报表

1. 打开报表详情页面
2. 点击**"编辑"**
3. 修改报表配置
4. 点击**"保存"**
   ```
   URL: /cms/report/storelletReport/{id} (GET/POST)
   ```

### 删除报表

1. 在报表列表中找到要删除的报表
2. 点击**"删除"**
   ```
   URL: /cms/report/storelletReport/{id} (DELETE)
   ```
3. 确认删除

报表查询使用 `StorelletReportRequestBody`：

## 🔢 优惠券计数（Count Coupon）

### 访问优惠券计数页面

1. 点击**"优惠券计数"**或**"Count Coupon"**
   ```
   URL: /cms/report/countCoupon
   ```

### 生成计数请求

1. 点击**"生成计数"**
2. 系统会生成一个唯一的计数 ID
   ```
   URL: /cms/report/countCoupon/{id}
   ```
3. 计数 ID 用于查询和验证

### 计数类型

CouponCountRequestType 包括：

| 类型 | 说明 |
|------|------|
| **TOTAL** | 总优惠券数 |
| **AVAILABLE** | 可用优惠券数 |
| **USED** | 已使用优惠券数 |
| **EXPIRED** | 已过期优惠券数 |

### 计数响应

CouponCountResponse 包含：

```javascript
{
  "id": "count123",
  "nonce": "unique-nonce-value",
  "favourId": 456,
  "countType": "AVAILABLE",
  "count": 1000,
  "timestamp": "2026-01-15T12:00:00Z"
}
```

### 计数缓存

系统使用缓存机制：
- **缓存时间**：2 分钟
- **缓存键**：基于 nonce
- **用途**：防止重复请求

### 使用场景

**实时查询**：
- 查询当前可用优惠券数量
- 不需要精确到实时的场景

**库存检查**：
- 发放前检查优惠券库存
- 避免超发

**数据验证**：
- 验证数据一致性
- 对比不同数据源

## 🎯 报表分析技巧

### 趋势分析

#### 时间序列分析

**查看增长趋势**：
- 选择较长的时间范围（如 3-6 个月）
- 观察指标的上升或下降趋势
- 识别季节性变化

**对比分析**：
- 同比对比：今年 vs 去年同期
- 环比对比：本月 vs 上月
- 活动对比：活动期间 vs 平时

### 漏斗分析

#### 优惠券漏斗

分析优惠券从发放到使用的完整流程：

```
发放数 → 领取数 → 使用数 → 兑换数
```

**计算各阶段转化率**：
- **领取率** = 领取数 / 发放数
- **使用率** = 使用数 / 领取数
- **兑换率** = 兑换数 / 使用数

**优化点**：
- 领取率低：优化推送时间和内容
- 使用率低：优化优惠券吸引力
- 兑换率低：优化使用流程

### 对比分析

#### 品牌对比

1. 选择多个品牌
2. 查看同一指标在不同品牌的表现
3. 识别表现好和表现差的环节
4. 学习优秀品牌的经验

#### 优惠券类型对比

1. 对比不同类型优惠券的表现
2. 分析：
   - 欢迎礼包 vs 印花优惠券
   - 推送优惠券 vs 手动发放
3. 找出最有效的优惠券类型

### 异常检测

#### 识别异常数据

**查看突然的峰值或谷值**：
- 检查是否有特殊事件
- 检查是否有系统问题
- 检查数据是否正确

**常见异常**：
- 使用率突然上升：可能是系统问题
- 发放数为 0：可能是配置错误
- 过敏率异常：可能是有效期设置问题

## 🔧 常见问题

### Q: 报表数据不准确？

**A**：
1. 检查选择的日期范围是否正确
2. 确认选择的品牌和店铺是否正确
3. 尝试刷新报表数据
4. 检查是否有数据延迟（通常 < 24 小时）
5. 联系技术支持验证数据源

### Q: 导出报表失败？

**A**：
1. 检查数据量是否过大（超过 10 万行）
2. 尝试缩小日期范围
3. 使用更具体的筛选条件
4. 尝试导出 CSV 而不是 Excel
5. 联系技术支持

### Q: 报表加载慢？

**A**：
1. 减少查询的时间范围
2. 使用更具体的筛选条件
3. 避免在高峰时段查看大量数据
4. 清除浏览器缓存
5. 使用性能更好的浏览器（Chrome）

### Q: 如何对比不同时期的数据？

**A**：
1. 导出不同时期的数据
2. 在 Excel 中进行对比分析
3. 或创建自定义报表配置
4. 使用数据透视表进行多维度对比

### Q: 报表中的数字和实际不符？

**A**：
1. 确认报表的计算逻辑
2. 检查数据的时间范围
3. 确认是否包含所有品牌/店铺
4. 检查是否有实时数据延迟
5. 联系技术支持验证数据源
