# CMS 報表與分析指南

## 標簽
#user-guide #report #analytics #advanced

## 概述

本文檔介紹如何使用 Storellet CMS 的報表和分析功能，包括數據概覽、各類報表查看、數據導出和自定義報表。

---

## 📊 訪問報表系統

### 步驟 1：進入報表頁面

1. 登錄 CMS 系統
2. 點擊頂部導航的**"報表"**或**"Report"**
3. 系統會跳轉到報表首頁
   ```
   URL: /cms/report/index
   ```

### 步驟 2：選擇品牌和日期范圍

在報表頁面：
- **品牌**：選擇要查看的品牌
- **開始日期**：選擇報表起始日期
- **結束日期**：選擇報表結束日期

---

## 📈 報表概覽（Overview）

### 訪問概覽頁面

1. 點擊**"概覽"**或**"Overview"**
2. 系統會顯示關鍵指標的匯總數據
   ```
   URL: /cms/report/overview
   ```

### 概覽顯示內容

#### 1. 折扣卡數據（Discount Cards）

顯示所有折扣卡的統計信息：

| 指標 | 說明 |
|------|------|
| **期間領取數** | 選擇時間段內領取折扣卡的用戶數 |
| **總領取數** | 從開始到現在的總領取數 |
| **領取率** | 領取用戶數 / 總用戶數 |

**查看方式**：
- 顯示所有活躍的折扣卡
- 每個折扣卡顯示期間領取數和總領取數
- 可以比較不同折扣卡的表現

#### 2. 歡迎禮包優惠券（Welcome Pack Coupons）

新用戶注冊時自動獲得的優惠券：

| 指標 | 說明 |
|------|------|
| **期間發放數** | 選擇時間段內發放的優惠券數量 |
| **總發放數** | 從開始到現在的總發放數 |
| **期間使用數** | 選擇時間段內使用的優惠券數量 |
| **總使用數** | 從開始到現在的總使用數 |
| **期間過期數** | 選擇時間段內過期的優惠券數量 |
| **總過期數** | 從開始到現在的總過期數 |

**計算公式**：
- **使用率** = 使用數 / 發放數
- **過期率** = 過期數 / 發放數

**用途**：
- 評估歡迎禮包的吸引力
- 分析新用戶的首次使用行為
- 優化歡迎禮包內容

#### 3. 印花優惠券（Stamp Coupons）

通過收集印花獲得的優惠券：

| 指標 | 說明 |
|------|------|
| **期間發放數** | 選擇時間段內完成印花卡並發放的優惠券 |
| **總發放數** | 總發放的印花優惠券數量 |
| **期間使用數** | 選擇時間段內使用的印花優惠券 |
| **總使用數** | 總使用的印花優惠券數 |
| **期間過期數** | 選擇時間段內過期的印花優惠券 |
| **總過期數** | 總過期的印花優惠券數 |

**分析要點**：
- 印花卡完成率
- 印花優惠券的使用率
- 優惠券過期率

#### 4. 推送優惠券（Push Coupons）

通過推送消息發放的優惠券：

| 指標 | 說明 |
|------|------|
| **期間發放數** | 選擇時間段內通過推送發放的優惠券 |
| **總發放數** | 總推送發放的優惠券數 |
| **期間使用數** | 選擇時間段內使用的推送優惠券 |
| **總使用數** | 總使用的推送優惠券數 |
| **期間過期數** | 選擇時間段內過期的推送優惠券 |
| **總過期數** | 總過期的推送優惠券數 |

**分析要點**：
- 推送效果分析
- 推送優惠券的吸引力
- 推送時間和內容優化

### 概覽最佳實踐

#### 數據對比分析

**時間對比**：
- 選擇不同時間段對比數據
- 例如：本月 vs 上月
- 例如：活動期間 vs 平時

**品牌對比**：
- 切換不同品牌查看數據
- 對比同一時期不同品牌的表現

**指標分析**：
- 關注使用率（使用數 / 發放數）
- 關注過期率（過期數 / 發放數）
- 使用率過低：可能是優惠券不夠吸引
- 過敏率過高：可能是有效期太短

## 📥 導出報表

### 導出步驟

1. 在報表頁面
2. 設置好篩選條件
3. 點擊**"導出"**或**"Export"**按鈕
   ```
   URL: /cms/report/{type}/{reqBrandId}/export
   ```
4. 選擇導出格式：
   - **Excel** (.xlsx) - 推薦
   - **CSV** - 用於數據分析

### 導出內容

導出的數據包含：

**基本信息**：
- 報表名稱
- 日期范圍
- 品牌信息
- 導出時間

**數據表格**：
- 根據報表類型包含不同字段
- 包含所有篩選後的數據
- 數據格式化處理

**匯總統計**：
- 總數統計
- 百分比計算
- 趨勢分析

### 導出最佳實踐

#### 文件命名

建議的命名格式：
```
[報表名稱]_[品牌]_[開始日期]_[結束日期].xlsx
例如：
用戶報表_BrandA_20260101_20260131.xlsx
優惠券使用統計_BrandB_202601.xlsx
```

#### 數據分析

**Excel 分析**：
- 使用數據透視表分析
- 創建圖表可視化
- 計算衍生指標

**定期導出**：
- 每月導出關鍵報表
- 保存歷史數據對比
- 建立數據趨勢分析

## 🔔 Coown 報警管理

### 什麼是 Coown 報警

Coown 報警是一種監控和通知機制，用於在特定事件或條件觸發時發送通知。

### 查看報警列表

1. 點擊**"Coown 報警"**或**"Coown Alarm"**
2. 系統會顯示所有報警配置
   ```
   URL: /cms/report/coownAlarm
   ```

### 報警信息

每個報警配置包含：
- **報警名稱**：報警的標識
- **執行時間**：報警觸發的默認時間
- **狀態**：啟用/禁用
- **配置詳情**：報警的具體配置

### 創建報警

1. 點擊**"新建報警"**
2. 填寫報警信息：
   - 報警名稱
   - 執行時間
   - 報警條件
   - 通知方式
3. 點擊**"保存"**
   ```
   URL: /cms/report/coownAlarm/{id}
   ```

### 編輯報警

1. 在報警列表中找到要編輯的報警
2. 點擊報警名稱進入詳情頁面
   ```
   URL: /cms/report/coownAlarm/{id}
   ```
3. 修改報警配置
4. 點擊**"保存"**

### 刪除報警

1. 在報警列表中找到要刪除的報警
2. 點擊**"刪除"**
   ```
   URL: /cms/report/coownAlarm/{id}/delete
   ```
3. 確認刪除

### 報警執行時間

默認執行時間（DEFAULT_EXECUTION_TIME）：
- 系統會在設定的時間檢查報警條件
- 如果條件滿足，發送報警通知
- 可以自定義每個報警的執行時間

- 選擇要顯示的指標
- 設置指標的計算方式

1. 點擊**"保存"**

## 🔢 優惠券計數（Count Coupon）

### 訪問優惠券計數頁面

1. 點擊**"優惠券計數"**或**"Count Coupon"**
   ```
   URL: /cms/report/countCoupon
   ```

### 生成計數請求

1. 點擊**"生成計數"**
2. 系統會生成一個唯一的計數 ID
   ```
   URL: /cms/report/countCoupon/{id}
   ```
3. 計數 ID 用於查詢和驗證

### 計數類型

CouponCountRequestType 包括：

| 類型 | 說明 |
|------|------|
| **TOTAL** | 總優惠券數 |
| **AVAILABLE** | 可用優惠券數 |
| **USED** | 已使用優惠券數 |
| **EXPIRED** | 已過期優惠券數 |

### 計數響應

CouponCountResponse 包含：

```javascript
{
  "id": "count123",
  "nonce": "unique-nonce-value",
  "favourId": 456,
  "countType": "AVAILABLE",
  "count": 1000,
  "timestamp": "2026-01-15T12:00:00Z"
}
```

### 計數緩存

系統使用緩存機制：
- **緩存時間**：2 分鐘
- **緩存鍵**：基於 nonce
- **用途**：防止重復請求

### 使用場景

**實時查詢**：
- 查詢當前可用優惠券數量
- 不需要精確到實時的場景

**庫存檢查**：
- 發放前檢查優惠券庫存
- 避免超發

**數據驗證**：
- 驗證數據一致性
- 對比不同數據源

## 🎯 報表分析技巧

### 趨勢分析

#### 時間序列分析

**查看增長趨勢**：
- 選擇較長的時間范圍（如 3-6 個月）
- 觀察指標的上升或下降趨勢
- 識別季節性變化

**對比分析**：
- 同比對比：今年 vs 去年同期
- 環比對比：本月 vs 上月
- 活動對比：活動期間 vs 平時

### 漏斗分析

#### 優惠券漏斗

分析優惠券從發放到使用的完整流程：

```
發放數 → 領取數 → 使用數 → 兌換數
```

**計算各階段轉化率**：
- **領取率** = 領取數 / 發放數
- **使用率** = 使用數 / 領取數
- **兌換率** = 兌換數 / 使用數

**優化點**：
- 領取率低：優化推送時間和內容
- 使用率低：優化優惠券吸引力
- 兌換率低：優化使用流程

### 對比分析

#### 品牌對比

1. 選擇多個品牌
2. 查看同一指標在不同品牌的表現
3. 識別表現好和表現差的環節
4. 學習優秀品牌的經驗

#### 優惠券類型對比

1. 對比不同類型優惠券的表現
2. 分析：
   - 歡迎禮包 vs 印花優惠券
   - 推送優惠券 vs 手動發放
3. 找出最有效的優惠券類型

### 異常檢測

#### 識別異常數據

**查看突然的峰值或谷值**：
- 檢查是否有特殊事件
- 檢查是否有系統問題
- 檢查數據是否正確

**常見異常**：
- 使用率突然上升：可能是系統問題
- 發放數為 0：可能是配置錯誤
- 過敏率異常：可能是有效期設置問題

## 🔧 常見問題

### Q: 報表數據不准確？

**A**：
1. 檢查選擇的日期范圍是否正確
2. 確認選擇的品牌和店鋪是否正確
3. 嘗試刷新報表數據
4. 檢查是否有數據延遲（通常 < 24 小時）
5. 聯系技術支持驗證數據源

### Q: 導出報表失敗？

**A**：
1. 檢查數據量是否過大（超過 10 萬行）
2. 嘗試縮小日期范圍
3. 使用更具體的篩選條件
4. 嘗試導出 CSV 而不是 Excel
5. 聯系技術支持

### Q: 報表加載慢？

**A**：
1. 減少查詢的時間范圍
2. 使用更具體的篩選條件
3. 避免在高峰時段查看大量數據
4. 清除瀏覽器緩存
5. 使用性能更好的瀏覽器（Chrome）

### Q: 如何對比不同時期的數據？

**A**：
1. 導出不同時期的數據
2. 在 Excel 中進行對比分析
3. 或創建自定義報表配置
4. 使用數據透視表進行多維度對比

### Q: 報表中的數字和實際不符？

**A**：
1. 確認報表的計算邏輯
2. 檢查數據的時間范圍
3. 確認是否包含所有品牌/店鋪
4. 檢查是否有實時數據延遲
5. 聯系技術支持驗證數據源

### 數據分析流程

#### 1. 明確分析目標

在查看報表前明確：
- 要解決什麼問題？
- 要分析哪個指標？
- 要對比什麼數據？

#### 2. 選擇合適的報表

根據分析目標選擇：
- 用戶增長 → 用戶報表
- 優惠券效果 → 優惠券報表
- 積分使用 → 積分報表

#### 3. 設置篩選條件

選擇合適的時間范圍和篩選條件：
- 足夠長的時間范圍看清趨勢
- 具體的篩選條件縮小范圍

#### 4. 分析和解讀

從數據中發現：
- 趨勢變化
- 異常情況
- 優化機會

#### 5. 制定行動計劃

根據數據分析結果：
- 優化產品設置
- 調整營銷策略
- 改進用戶體驗
