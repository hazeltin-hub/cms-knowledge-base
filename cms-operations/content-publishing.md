# 內容發布指南

## 標簽
`#operation` `#workflow` `#publishing`

## 概述

本文檔介紹 Storellet CMS 的內容發布流程和最佳實踐。

## 發布流程

### 完整工作流

```
創建草稿 → 編輯內容 → 提交審核 → 審核批准 → 發布 → 多渠道分發
```

### 詳細步驟

#### 1. 創建草稿

```javascript
// 通過 API 創建草稿
POST /api/content
{
  "title": "文章標題",
  "contentType": "article",
  "content": "文章內容...",
  "language": "zh-CN",
  "status": "draft"
}
```

#### 2. 編輯內容

- 使用 AI 寫作助手優化內容
- 檢查 SEO 分數
- 添加標簽和分類
- 設置發布時間（可選）

#### 3. 提交審核

```javascript
// 提交審核
PUT /api/content/{id}/submit
```

#### 4. 審核批准

審核員會檢查：
- 內容質量和准確性
- SEO 優化是否充分
- 多語言翻譯是否完整
- 元數據是否正確

#### 5. 發布內容

```javascript
// 發布內容
PUT /api/content/{id}/publish
{
  "channels": ["website", "mobile-app", "social-media"]
}
```

## 多渠道發布

### 支持的渠道

| 渠道 | 自動發布 | 需要配置 |
|------|----------|----------|
| 網站 | ✓ | ✓ |
| 移動 App | ✓ | ✓ |
| 社交媒體 | ✓ | ✓ |
| Email | ✓ | ✓ |

### 發布配置

在 CMS 設置中配置各渠道的 API 密鑰和發布格式。

## SEO 最佳實踐

發布前檢查：
- ✓ 標題長度 50-60 字符
- ✓ Meta 描述 150-160 字符
- ✓ 關鍵詞密度 1-2%
- ✓ 內容長度 > 300 字
- ✓ 可讀性分數 > 60

## 常見問題

### Q: 如何定時發布內容？

1. 在編輯頁面找到"發布設置"
2. 選擇"定時發布"
3. 設置發布日期和時間
4. 系統會在指定時間自動發布

### Q: 發布後可以修改嗎？

可以，但需要：
1. 創建新版本
2. 重新提交審核
3. 審核通過後更新發布

### Q: 如何撤回已發布的內容？

1. 進入內容管理頁面
2. 找到已發布的內容
3. 點擊"撤回"或"設為草稿"
