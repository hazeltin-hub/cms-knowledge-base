# 内容发布指南

## 标签
`#operation` `#workflow` `#publishing`

## 概述

本文档介绍 Storellet CMS 的内容发布流程和最佳实践。

## 发布流程

### 完整工作流

```
创建草稿 → 编辑内容 → 提交审核 → 审核批准 → 发布 → 多渠道分发
```

### 详细步骤

#### 1. 创建草稿

```javascript
// 通过 API 创建草稿
POST /api/content
{
  "title": "文章标题",
  "contentType": "article",
  "content": "文章内容...",
  "language": "zh-CN",
  "status": "draft"
}
```

#### 2. 编辑内容

- 使用 AI 写作助手优化内容
- 检查 SEO 分数
- 添加标签和分类
- 设置发布时间（可选）

#### 3. 提交审核

```javascript
// 提交审核
PUT /api/content/{id}/submit
```

#### 4. 审核批准

审核员会检查：
- 内容质量和准确性
- SEO 优化是否充分
- 多语言翻译是否完整
- 元数据是否正确

#### 5. 发布内容

```javascript
// 发布内容
PUT /api/content/{id}/publish
{
  "channels": ["website", "mobile-app", "social-media"]
}
```

### 发布配置

在 CMS 设置中配置各渠道的 API 密钥和发布格式。

## SEO 最佳实践

发布前检查：
- ✓ 标题长度 50-60 字符
- ✓ Meta 描述 150-160 字符
- ✓ 关键词密度 1-2%
- ✓ 内容长度 > 300 字
- ✓ 可读性分数 > 60

## 常见问题

### Q: 如何定时发布内容？

1. 在编辑页面找到"发布设置"
2. 选择"定时发布"
3. 设置发布日期和时间
4. 系统会在指定时间自动发布

### Q: 发布后可以修改吗？

可以，但需要：
1. 创建新版本
2. 重新提交审核
3. 审核通过后更新发布

### Q: 如何撤回已发布的内容？

1. 进入内容管理页面
2. 找到已发布的内容
3. 点击"撤回"或"设为草稿"
