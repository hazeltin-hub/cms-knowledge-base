# CMS 推送管理指南

## 标签
#user-guide #push #marketing #intermediate

## 概述

本文档介绍如何使用 Storellet CMS 的推送功能，包括优惠券推送、积分批量调整、推送通知、新闻推送和批量操作。

---

## 📱 访问推送管理

### 步骤 1：进入推送页面

1. 登录 CMS 系统
2. 点击顶部导航的**"推送"**或**"Push"**
3. 系统会跳转到推送管理首页
   ```
   URL: /cms/push/index
   ```

### 步骤 2：选择推送类型

推送管理包括以下功能：
- **优惠券推送**：向用户发放优惠券
- **积分批量调整**：批量调整用户积分
- **推送通知**：发送推送通知消息
- **新闻推送**：发送新闻和公告
- **批量操作**：批量优惠券和积分操作

---

## 🎫 优惠券推送

### 什么是优惠券推送

优惠券推送用于向符合条件的用户批量发放优惠券，支持：
- 按品牌发放
- 按用户筛选发放
- 定时发放
- 自定义用户列表发放

### 访问优惠券推送

1. 在推送管理页面
2. 点击**"优惠券推送"**或**"Push Coupon"**
3. 系统会显示所有优惠券推送任务
   ```
   URL: /cms/push/coupon
   ```

### 查看推送列表

显示所有优惠券推送任务：

| 字段 | 说明 |
|------|------|
| **推送 ID** | 推送任务唯一标识 |
| **品牌** | 推送所属品牌 |
| **优惠券** | 推送的优惠券类型 |
| **数量** | 推送数量 |
| **接收者类型** | 所有用户/筛选用户/自定义用户 |
| **推送时间** | 计划推送时间 |
| **状态** | 待推送/已推送/已取消 |
| **操作** | 编辑、删除 |

### 创建优惠券推送

#### 步骤 1：点击新建

1. 在优惠券推送列表页面
2. 点击**"新建推送"**或**"Add"**
   ```
   URL: /cms/push/coupon/add
   ```

#### 步骤 2：基本信息

**品牌选择**：
- 选择要推送的品牌
- 推送只会发送给该品牌的用户

**优惠券选择**：
- 从下拉菜单选择优惠券类型
- 确认优惠券库存充足
- 确认优惠券在有效期内

**推送数量**：
- 输入要推送的优惠券数量
- 例如：1000 表示推送 1000 张优惠券
- ⚠️ 不能超过优惠券库存

#### 步骤 3：接收者配置

选择接收者类型：

**类型 1：所有用户（All Users）**

- 推送给品牌的所有用户
- 无需额外配置
- 适用于通用优惠活动

**类型 2：筛选用户（Filtered Users）**

根据条件筛选用户：

| 筛选条件 | 说明 |
|---------|------|
| **必需折扣卡** | 用户必须拥有指定的折扣卡 |
| **排除折扣卡** | 排除拥有指定折扣卡的用户 |
| **必需全部** | 用户必须拥有所有必需的折扣卡（或只拥有其一） |
| **会员加入日期** | 只推送给在指定日期前加入的会员 |
| **优先推送** | 启用优先推送功能 |

**配置步骤**：

1. **必需折扣卡**：
   - 从列表选择折扣卡
   - 可选择多个
   - 用户必须拥有至少一张（或全部，取决于"必需全部"设置）

2. **排除折扣卡**：
   - 选择要排除的用户折扣卡
   - 拥有这些折扣卡的用户不会收到推送

3. **会员加入日期**：
   - 选择日期
   - 只推送给在该日期前加入的会员
   - 例如：2026-01-01 表示只推送给 2026年1月1日前加入的会员

4. **优先推送**：
   - 启用后，优先推送给高价值用户
   - 基于用户活跃度和消费金额

**类型 3：自定义用户（Custom Users）**

上传 CSV 文件指定接收用户：

**CSV 文件格式**：
```csv
memberId
123456789
987654321
```

**字段说明**：
- 第一行：`memberId`（固定）
- 后续行：会员 ID
- 每行一个会员 ID

**上传步骤**：
1. 准备 CSV 文件
2. 点击**"选择文件"**
3. 选择 CSV 文件
4. 系统会验证文件格式
5. 显示用户数量

**注意事项**：
- ⚠️ 确保会员 ID 正确
- ⚠️ 文件大小不超过 10MB
- ⚠️ 最多 10 万用户

#### 步骤 4：推送时间配置

**计划推送时间**：
- **发布日期**：选择推送日期
- **发布时间**：选择推送时间（HH:mm）
- **创建日期**：默认当前时间

**定时推送**：
- ✅ 设置未来的推送时间
- ✅ 系统会在指定时间自动推送
- ⚠️ 推送时间不能早于当前时间

**立即推送**：
- ✅ 选择当前时间
- ✅ 点击保存后立即推送

#### 步骤 5：高级选项

**绕过第三方事件追踪**：
- 启用后，推送不会触发第三方事件追踪
- 用于测试或特殊场景

#### 步骤 6：保存推送

1. 点击**"保存"**或**"Submit"**
   ```
   URL: /cms/push/coupon/{id} (POST with operation=submit)
   ```
2. 系统会验证配置
3. 保存成功后显示在推送列表

### 编辑优惠券推送

1. 在推送列表中找到要编辑的推送
2. 点击推送 ID 进入编辑页面
   ```
   URL: /cms/push/coupon/{id}
   ```
3. 修改推送配置
4. 点击**"保存"**

**注意事项**：
- ⚠️ 已推送的任务不能编辑
- ⚠️ 只能编辑待推送的任务
- ⚠️ 修改接收者会重新计算用户数量

### 删除优惠券推送

1. 在推送列表中找到要删除的推送
2. 点击**"删除"**
   ```
   URL: /cms/push/coupon/{id}/remove
   ```
3. 确认删除

**注意事项**：
- ⚠️ 已推送的任务不能删除
- ⚠️ 只能删除待推送的任务
- ⚠️ 删除操作不可逆

## 💰 积分批量调整

### 什么是积分批量调整

积分批量调整用于通过 CSV 文件批量增加或扣减用户积分，适用于：
- 批量补偿积分
- 活动奖励发放
- 数据纠错

### 访问积分批量调整

1. 在推送管理页面
2. 点击**"积分批量调整"**或**"Batch Point Adjustment"**
3. 系统会显示所有批量调整任务
   ```
   URL: /cms/push/batchPointAdjustment
   ```

### 查看批量调整列表

显示所有积分批量调整任务：

| 字段 | 说明 |
|------|------|
| **任务 ID** | 批量调整任务唯一标识 |
| **品牌** | 所属品牌 |
| **创建时间** | 任务创建时间 |
| **计划时间** | 计划执行时间 |
| **状态** | 待执行/执行中/已完成/失败 |
| **操作** | 查看详情 |

### 创建批量调整任务

#### 步骤 1：点击新建

1. 在批量调整列表页面
2. 点击**"新建任务"**
   ```
   URL: /cms/push/batchPointAdjustment/add
   ```

#### 步骤 2：基本信息

**品牌选择**：
- 选择要调整积分的品牌

**计划执行时间**：
- **执行日期**：选择执行日期
- **执行时间**：选择执行时间（HH:mm）
- 系统会在指定时间自动执行

#### 步骤 3：准备 CSV 文件

**CSV 文件格式**：
```csv
memberId,point,expiryDate
123456789,100,20261231
987654321,-50,
```

**字段说明**：

| 字段 | 说明 | 必填 | 示例 |
|------|------|------|------|
| **memberId** | 会员 ID | ✅ | 123456789 |
| **point** | 积分数量（正数=增加，负数=扣减） | ✅ | 100 |
| **expiryDate** | 积分过期日期（可选，格式：yyyyMMdd） | ❌ | 20261231 |

**CSV 示例**：

```csv
memberId,point,expiryDate
123456789,100,20261231
987654321,200,20271231
555555555,-50,
```

**说明**：
- 第 1 行：增加 100 积分，2026年12月31日过期
- 第 2 行：增加 200 积分，2027年12月31日过期
- 第 3 行：扣减 50 积分，无过期日期

#### 步骤 4：上传 CSV 文件

1. 点击**"选择文件"**
2. 选择准备好的 CSV 文件
3. 系统会验证文件：
   - 格式是否正确
   - 会员 ID 是否有效
   - 积分数量是否有效
4. 显示解析结果

#### 步骤 5：确认创建

1. 查看解析结果
2. 确认用户数量和积分数量
3. 点击**"创建"**
   ```
   URL: /cms/push/batchPointAdjustment/add (POST)
   ```
4. 系统会创建批量调整任务

### 查看任务详情

1. 在批量调整列表中找到任务
2. 点击任务 ID 查看详情
3. 可以查看：
   - 执行状态
   - 成功/失败记录
   - 错误信息

### 任务状态

| 状态 | 说明 |
|------|------|
| **待执行** | 任务已创建，等待执行时间到达 |
| **执行中** | 正在执行积分调整 |
| **已完成** | 任务执行完成 |
| **失败** | 任务执行失败 |

显示所有推送通知：

| 字段 | 说明 |

| **通知 ID** | 通知唯一标识 |
| **品牌** | 所属品牌 |

| **内容** | 通知内容（摘要） |

1. 在通知列表中找到要删除的通知

## 🎯 推送最佳实践

### 优惠券推送

#### 推送时机

**最佳推送时间**：
- 工作日：10:00-11:00 或 15:00-16:00
- 周末：11:00-12:00
- 避免在深夜或凌晨推送

**推送频率**：
- ✅ 同一用户每周不超过 2-3 次
- ✅ 重要活动可适当增加
- ❌ 避免每天推送导致用户反感

#### 接收者筛选

**精准推送**：
- ✅ 根据用户行为筛选
- ✅ 根据购买历史筛选
- ✅ 根据会员等级筛选

**避免推送疲劳**：
- ✅ 排除最近收过推送的用户
- ✅ 使用"会员加入日期"避免推送给新用户

#### 内容优化

**优惠券选择**：
- ✅ 选择吸引力高的优惠券
- ✅ 确保优惠券与用户匹配
- ✅ 设置合理的有效期

**推送测试**：
- ✅ 先小范围测试
- ✅ 检查推送效果
- ✅ 优化后大规模推送

### 积分批量调整

#### 数据验证

**调整前检查**：
- ✅ 验证 CSV 文件格式
- ✅ 确认会员 ID 正确
- ✅ 确认积分数量合理
- ✅ 设置合理的过期日期

**测试执行**：
- ✅ 先用少量数据测试
- ✅ 确认结果符合预期
- ✅ 再执行大规模调整

#### 执行时间

**选择合适的执行时间**：
- ✅ 避开高峰时段
- ✅ 选择系统负载低的时间
- ✅ 建议凌晨或清晨执行

### 推送通知

#### 内容优化

**标题**：
- ✅ 简洁明了（不超过 20 字）
- ✅ 突出重点信息
- ✅ 吸引点击

**内容**：
- ✅ 清晰说明通知目的
- ✅ 包含行动号召（CTA）
- ✅ 避免过长（不超过 100 字）

**格式建议**：
```
【限时优惠】您的专属优惠券已到账！
尊敬的会员，恭喜您获得一张满100减20优惠券，
有效期至本月底，点击查看详情~
```

## 🔧 常见问题

### Q: 优惠券推送失败？

**A**：
1. 检查优惠券库存是否充足
2. 确认优惠券在有效期内
3. 检查接收者配置是否正确
4. 查看错误日志了解具体原因
5. 联系技术支持

### Q: CSV 文件上传失败？

**A**：
1. 检查文件格式是否为 CSV
2. 确认文件编码为 UTF-8
3. 验证字段名称和顺序
4. 检查数据是否有效
5. 确认文件大小不超过限制

### Q: 批量积分调整失败？

**A**：
1. 检查会员 ID 是否正确
2. 确认积分数量格式正确
3. 验证过期日期格式（yyyyMMdd）
4. 查看错误日志
5. 修复 CSV 后重新提交

### Q: 推送通知用户收不到？

**A**：
1. 确认用户已启用推送通知
2. 检查用户设备是否在线
3. 验证推送时间是否正确
4. 查看推送日志
5. 联系技术支持

### Q: 如何查看推送效果？

**A**：
1. 在报表页面查看优惠券使用统计
2. 查看推送后的用户活跃度
3. 分析优惠券转化率
4. 对比不同推送策略的效果

### Q: 推送给自定义用户时，CSV 文件格式错误？

**A**：

**正确的格式**：
```csv
memberId
123456789
987654321
```

**常见错误**：
- ❌ 缺少表头（第一行必须是 `memberId`）
- ❌ 使用其他字段名（必须是 `memberId`）
- ❌ 包含空行
- ❌ 编码不是 UTF-8
