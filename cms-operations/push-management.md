# CMS 推送管理指南

## 標簽
#user-guide #push #marketing #intermediate

## 概述

本文檔介紹如何使用 Storellet CMS 的推送功能，包括優惠券推送、積分批量調整、推送通知、新聞推送和批量操作。

---

## 📱 訪問推送管理

### 步驟 1：進入推送頁面

1. 登錄 CMS 系統
2. 點擊頂部導航的**"推送"**或**"Push"**
3. 系統會跳轉到推送管理首頁
   ```
   URL: /cms/push/index
   ```

### 步驟 2：選擇推送類型

推送管理包括以下功能：
- **優惠券推送**：向用戶發放優惠券
- **積分批量調整**：批量調整用戶積分
- **推送通知**：發送推送通知消息
- **新聞推送**：發送新聞和公告
- **批量操作**：批量優惠券和積分操作

---

## 🎫 優惠券推送

### 什麼是優惠券推送

優惠券推送用於向符合條件的用戶批量發放優惠券，支持：
- 按品牌發放
- 按用戶篩選發放
- 定時發放
- 自定義用戶列表發放

### 訪問優惠券推送

1. 在推送管理頁面
2. 點擊**"優惠券推送"**或**"Push Coupon"**
3. 系統會顯示所有優惠券推送任務
   ```
   URL: /cms/push/coupon
   ```

### 查看推送列表

顯示所有優惠券推送任務：

| 字段 | 說明 |
|------|------|
| **推送 ID** | 推送任務唯一標識 |
| **品牌** | 推送所屬品牌 |
| **優惠券** | 推送的優惠券類型 |
| **數量** | 推送數量 |
| **接收者類型** | 所有用戶/篩選用戶/自定義用戶 |
| **推送時間** | 計劃推送時間 |
| **狀態** | 待推送/已推送/已取消 |
| **操作** | 編輯、刪除 |

### 創建優惠券推送

#### 步驟 1：點擊新建

1. 在優惠券推送列表頁面
2. 點擊**"新建推送"**或**"Add"**
   ```
   URL: /cms/push/coupon/add
   ```

#### 步驟 2：基本信息

**品牌選擇**：
- 選擇要推送的品牌
- 推送只會發送給該品牌的用戶

**優惠券選擇**：
- 從下拉菜單選擇優惠券類型
- 確認優惠券庫存充足
- 確認優惠券在有效期內

**推送數量**：
- 輸入要推送的優惠券數量
- 例如：1000 表示推送 1000 張優惠券
- ⚠️ 不能超過優惠券庫存

#### 步驟 3：接收者配置

選擇接收者類型：

**類型 1：所有用戶（All Users）**

- 推送給品牌的所有用戶
- 無需額外配置
- 適用於通用優惠活動

**類型 2：篩選用戶（Filtered Users）**

根據條件篩選用戶：

| 篩選條件 | 說明 |
|---------|------|
| **必需折扣卡** | 用戶必須擁有指定的折扣卡 |
| **排除折扣卡** | 排除擁有指定折扣卡的用戶 |
| **必需全部** | 用戶必須擁有所有必需的折扣卡（或只擁有其一） |
| **會員加入日期** | 只推送給在指定日期前加入的會員 |
| **優先推送** | 啟用優先推送功能 |

**配置步驟**：

1. **必需折扣卡**：
   - 從列表選擇折扣卡
   - 可選擇多個
   - 用戶必須擁有至少一張（或全部，取決於"必需全部"設置）

2. **排除折扣卡**：
   - 選擇要排除的用戶折扣卡
   - 擁有這些折扣卡的用戶不會收到推送

3. **會員加入日期**：
   - 選擇日期
   - 只推送給在該日期前加入的會員
   - 例如：2026-01-01 表示只推送給 2026年1月1日前加入的會員

4. **優先推送**：
   - 啟用後，優先推送給高價值用戶
   - 基於用戶活躍度和消費金額

**類型 3：自定義用戶（Custom Users）**

上傳 CSV 文件指定接收用戶：

**CSV 文件格式**：
```csv
memberId
123456789
987654321
```

**字段說明**：
- 第一行：`memberId`（固定）
- 後續行：會員 ID
- 每行一個會員 ID

**上傳步驟**：
1. 准備 CSV 文件
2. 點擊**"選擇文件"**
3. 選擇 CSV 文件
4. 系統會驗證文件格式
5. 顯示用戶數量

**注意事項**：
- ⚠️ 確保會員 ID 正確
- ⚠️ 文件大小不超過 10MB
- ⚠️ 最多 10 萬用戶

#### 步驟 4：推送時間配置

**計劃推送時間**：
- **發布日期**：選擇推送日期
- **發布時間**：選擇推送時間（HH:mm）
- **創建日期**：默認當前時間

**定時推送**：
- ✅ 設置未來的推送時間
- ✅ 系統會在指定時間自動推送
- ⚠️ 推送時間不能早於當前時間

**立即推送**：
- ✅ 選擇當前時間
- ✅ 點擊保存後立即推送

#### 步驟 5：高級選項

**繞過第三方事件追蹤**：
- 啟用後，推送不會觸發第三方事件追蹤
- 用於測試或特殊場景

#### 步驟 6：保存推送

1. 點擊**"保存"**或**"Submit"**
   ```
   URL: /cms/push/coupon/{id} (POST with operation=submit)
   ```
2. 系統會驗證配置
3. 保存成功後顯示在推送列表

### 編輯優惠券推送

1. 在推送列表中找到要編輯的推送
2. 點擊推送 ID 進入編輯頁面
   ```
   URL: /cms/push/coupon/{id}
   ```
3. 修改推送配置
4. 點擊**"保存"**

**注意事項**：
- ⚠️ 已推送的任務不能編輯
- ⚠️ 只能編輯待推送的任務
- ⚠️ 修改接收者會重新計算用戶數量

### 刪除優惠券推送

1. 在推送列表中找到要刪除的推送
2. 點擊**"刪除"**
   ```
   URL: /cms/push/coupon/{id}/remove
   ```
3. 確認刪除

**注意事項**：
- ⚠️ 已推送的任務不能刪除
- ⚠️ 只能刪除待推送的任務
- ⚠️ 刪除操作不可逆

## 💰 積分批量調整

### 什麼是積分批量調整

積分批量調整用於通過 CSV 文件批量增加或扣減用戶積分，適用於：
- 批量補償積分
- 活動獎勵發放
- 數據糾錯

### 訪問積分批量調整

1. 在推送管理頁面
2. 點擊**"積分批量調整"**或**"Batch Point Adjustment"**
3. 系統會顯示所有批量調整任務
   ```
   URL: /cms/push/batchPointAdjustment
   ```

### 查看批量調整列表

顯示所有積分批量調整任務：

| 字段 | 說明 |
|------|------|
| **任務 ID** | 批量調整任務唯一標識 |
| **品牌** | 所屬品牌 |
| **創建時間** | 任務創建時間 |
| **計劃時間** | 計劃執行時間 |
| **狀態** | 待執行/執行中/已完成/失敗 |
| **操作** | 查看詳情 |

### 創建批量調整任務

#### 步驟 1：點擊新建

1. 在批量調整列表頁面
2. 點擊**"新建任務"**
   ```
   URL: /cms/push/batchPointAdjustment/add
   ```

#### 步驟 2：基本信息

**品牌選擇**：
- 選擇要調整積分的品牌

**計劃執行時間**：
- **執行日期**：選擇執行日期
- **執行時間**：選擇執行時間（HH:mm）
- 系統會在指定時間自動執行

#### 步驟 3：准備 CSV 文件

**CSV 文件格式**：
```csv
memberId,point,expiryDate
123456789,100,20261231
987654321,-50,
```

**字段說明**：

| 字段 | 說明 | 必填 | 示例 |
|------|------|------|------|
| **memberId** | 會員 ID | ✅ | 123456789 |
| **point** | 積分數量（正數=增加，負數=扣減） | ✅ | 100 |
| **expiryDate** | 積分過期日期（可選，格式：yyyyMMdd） | ❌ | 20261231 |

**CSV 示例**：

```csv
memberId,point,expiryDate
123456789,100,20261231
987654321,200,20271231
555555555,-50,
```

**說明**：
- 第 1 行：增加 100 積分，2026年12月31日過期
- 第 2 行：增加 200 積分，2027年12月31日過期
- 第 3 行：扣減 50 積分，無過期日期

#### 步驟 4：上傳 CSV 文件

1. 點擊**"選擇文件"**
2. 選擇准備好的 CSV 文件
3. 系統會驗證文件：
   - 格式是否正確
   - 會員 ID 是否有效
   - 積分數量是否有效
4. 顯示解析結果

#### 步驟 5：確認創建

1. 查看解析結果
2. 確認用戶數量和積分數量
3. 點擊**"創建"**
   ```
   URL: /cms/push/batchPointAdjustment/add (POST)
   ```
4. 系統會創建批量調整任務

### 查看任務詳情

1. 在批量調整列表中找到任務
2. 點擊任務 ID 查看詳情
3. 可以查看：
   - 執行狀態
   - 成功/失敗記錄
   - 錯誤信息

### 任務狀態

| 狀態 | 說明 |
|------|------|
| **待執行** | 任務已創建，等待執行時間到達 |
| **執行中** | 正在執行積分調整 |
| **已完成** | 任務執行完成 |
| **失敗** | 任務執行失敗 |

## 🔔 推送通知

### 什麼是推送通知

推送通知用於向用戶發送實時通知消息，包括：
- 活動通知
- 促銷提醒
- 會員通知
- 系統消息

### 訪問推送通知

1. 在推送管理頁面
2. 點擊**"推送通知"**或**"Notification"**
3. 系統會顯示所有推送通知
   ```
   URL: /cms/push/notification
   ```

### 查看通知列表

顯示所有推送通知：

| 字段 | 說明 |
|------|------|
| **通知 ID** | 通知唯一標識 |
| **品牌** | 所屬品牌 |
| **標題** | 通知標題 |
| **內容** | 通知內容（摘要） |
| **發送時間** | 計劃發送時間 |
| **狀態** | 待發送/已發送/已取消 |
| **操作** | 編輯、刪除 |

### 創建推送通知

#### 步驟 1：點擊新建

1. 在推送通知列表頁面
2. 點擊**"新建通知"**

#### 步驟 2：填寫通知信息

**基本信息**：
- **品牌**：選擇要發送通知的品牌
- **標題**：輸入通知標題（建議不超過 20 字）
- **內容**：輸入通知內容（建議不超過 100 字）

**接收者配置**：
- **發送給所有用戶**：發送給品牌的所有用戶
- **發送給特定用戶**：上傳 CSV 文件指定用戶

**發送時間**：
- **立即發送**：保存後立即發送
- **定時發送**：設置發送日期和時間

#### 步驟 3：保存通知

1. 點擊**"保存"**
   ```
   URL: /cms/push/notification/{id} (POST)
   ```
2. 系統會創建通知

1. 在通知列表中找到要編輯的通知
2. 點擊通知 ID 進入編輯頁面
   ```
   URL: /cms/push/notification/{id}
   ```
3. 修改通知內容
4. 點擊**"保存"**

**注意事項**：
- ⚠️ 已發送的通知不能編輯
- ⚠️ 只能編輯待發送的通知

### 刪除推送通知

1. 在通知列表中找到要刪除的通知
2. 點擊**"刪除"**
   ```
   URL: /cms/push/notification/{id}/remove
   ```
3. 確認刪除

## 🎯 推送最佳實踐

### 優惠券推送

#### 推送時機

**最佳推送時間**：
- 工作日：10:00-11:00 或 15:00-16:00
- 周末：11:00-12:00
- 避免在深夜或凌晨推送

**推送頻率**：
- ✅ 同一用戶每周不超過 2-3 次
- ✅ 重要活動可適當增加
- ❌ 避免每天推送導致用戶反感

#### 接收者篩選

**精准推送**：
- ✅ 根據用戶行為篩選
- ✅ 根據購買歷史篩選
- ✅ 根據會員等級篩選

**避免推送疲勞**：
- ✅ 排除最近收過推送的用戶
- ✅ 使用"會員加入日期"避免推送給新用戶

#### 內容優化

**優惠券選擇**：
- ✅ 選擇吸引力高的優惠券
- ✅ 確保優惠券與用戶匹配
- ✅ 設置合理的有效期

**推送測試**：
- ✅ 先小范圍測試
- ✅ 檢查推送效果
- ✅ 優化後大規模推送

### 積分批量調整

#### 數據驗證

**調整前檢查**：
- ✅ 驗證 CSV 文件格式
- ✅ 確認會員 ID 正確
- ✅ 確認積分數量合理
- ✅ 設置合理的過期日期

**測試執行**：
- ✅ 先用少量數據測試
- ✅ 確認結果符合預期
- ✅ 再執行大規模調整

#### 執行時間

**選擇合適的執行時間**：
- ✅ 避開高峰時段
- ✅ 選擇系統負載低的時間
- ✅ 建議凌晨或清晨執行

### 推送通知

#### 內容優化

**標題**：
- ✅ 簡潔明了（不超過 20 字）
- ✅ 突出重點信息
- ✅ 吸引點擊

**內容**：
- ✅ 清晰說明通知目的
- ✅ 包含行動號召（CTA）
- ✅ 避免過長（不超過 100 字）

**格式建議**：
```
【限時優惠】您的專屬優惠券已到賬！
尊敬的會員，恭喜您獲得一張滿100減20優惠券，
有效期至本月底，點擊查看詳情~
```

## 🔧 常見問題

### Q: 優惠券推送失敗？

**A**：
1. 檢查優惠券庫存是否充足
2. 確認優惠券在有效期內
3. 檢查接收者配置是否正確
4. 查看錯誤日志了解具體原因
5. 聯系技術支持

### Q: CSV 文件上傳失敗？

**A**：
1. 檢查文件格式是否為 CSV
2. 確認文件編碼為 UTF-8
3. 驗證字段名稱和順序
4. 檢查數據是否有效
5. 確認文件大小不超過限制

### Q: 批量積分調整失敗？

**A**：
1. 檢查會員 ID 是否正確
2. 確認積分數量格式正確
3. 驗證過期日期格式（yyyyMMdd）
4. 查看錯誤日志
5. 修復 CSV 後重新提交

### Q: 推送通知用戶收不到？

**A**：
1. 確認用戶已啟用推送通知
2. 檢查用戶設備是否在線
3. 驗證推送時間是否正確
4. 查看推送日志
5. 聯系技術支持

### Q: 如何查看推送效果？

**A**：
1. 在報表頁面查看優惠券使用統計
2. 查看推送後的用戶活躍度
3. 分析優惠券轉化率
4. 對比不同推送策略的效果

### Q: 推送給自定義用戶時，CSV 文件格式錯誤？

**A**：

**正確的格式**：
```csv
memberId
123456789
987654321
```

**常見錯誤**：
- ❌ 缺少表頭（第一行必須是 `memberId`）
- ❌ 使用其他字段名（必須是 `memberId`）
- ❌ 包含空行
- ❌ 編碼不是 UTF-8
