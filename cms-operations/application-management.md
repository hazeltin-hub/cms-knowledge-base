# CMS 應用配置管理指南

## 標簽
#user-guide #application #settings #advanced

## 概述

本文檔介紹如何配置和管理 Storellet CMS 應用設置，包括前端頁面配置、圖片管理、標簽管理、地區設置、POS 密鑰管理和 PMS 集成。

---

## 🔧 訪問應用配置

### 步驟 1：進入應用配置頁面

1. 登錄 CMS 系統
2. 點擊頂部導航的**"應用"**或**"Application"**
3. 系統會跳轉到應用配置首頁
   ```
   URL: /cms/application/index
   ```

---

## 🎨 前端頁面設置（Front Page Settings）

### 什麼是前端頁面設置

前端頁面設置用於配置應用首頁的布局和顯示內容，包括：
- 輪播圖
- 特色項目
- 品牌分組
- 地理位置分組

### 訪問前端頁面設置

1. 在應用配置頁面
2. 點擊**"前端頁面設置"**或**"Front Page Setting"**
3. 系統會顯示當前的前端頁面配置
   ```
   URL: /cms/application/frontPageSetting
   ```

### 查看當前配置

頁面會顯示：

#### 1. 當前配置 JSON

顯示當前的頁面布局配置：
```json
[
  {
    "type": "banner",
    "items": [...]
  },
  {
    "type": "featured",
    "items": [...]
  }
]
```

#### 2. 品牌標簽組（Tag Groups）

顯示可用的品牌標簽：
- 標簽組名稱
- 標簽列表
- 標簽順序

#### 3. 地理標簽組（Geo Tag Groups）

顯示可用的地理標簽：
- 地理區域名稱
- 區域內的標簽

#### 4. 特色項目（Featured Items）

顯示當前的特色項目列表：
- 活動名稱
- 圖片
- 鏈接

### 修改前端頁面配置

#### 步驟 1：編輯 JSON 配置

1. 在 JSON 編輯器中修改配置
2. 配置類型包括：
   - **banner**：輪播圖配置
   - **featured**：特色項目配置
   - **tagGroup**：標簽分組配置

#### 步驟 2：配置輪播圖

```json
{
  "type": "banner",
  "items": [
    {
      "imageId": "image123",
      "link": "https://example.com",
      "title": "Banner 1"
    }
  ]
}
```

#### 步驟 3：配置特色項目

```json
{
  "type": "featured",
  "items": [
    {
      "campaignName": "活動名稱",
      "imageUrl": "https://example.com/image.jpg",
      "link": "campaign://123"
    }
  ]
}
```

#### 步驟 4：保存配置

1. 點擊**"保存"**或**"Update"**
   ```
   URL: /cms/application/frontPageSetting (POST)
   ```
2. 系統會：
   - 創建新的配置記錄
   - 將舊配置設為非活躍狀態
   - 新配置立即生效

### 配置最佳實踐

#### 輪播圖配置
- ✅ 使用高質量的圖片（建議 1080x1920 像素）
- ✅ 每張圖片添加描述性標題
- ✅ 設置正確的跳轉鏈接
- ❌ 不要添加過多輪播圖（建議 3-5 張）

#### 特色項目配置
- ✅ 選擇當前熱門的活動
- ✅ 定期更新內容
- ✅ 使用吸引人的圖片
- ❌ 不要顯示已結束的活動

## 🖼️ 圖片管理（Image Store）

### 什麼是圖片存儲

圖片存儲用於集中管理應用中使用的所有圖片，按類型分類：
- **banner**：輪播圖
- **featured**：特色項目圖片
- **brand**：品牌圖片
- **campaign**：活動圖片

### 訪問圖片管理

1. 在應用配置頁面
2. 選擇要管理的圖片類型
3. 系統會顯示該類型的所有圖片
   ```
   URL: /cms/application/imageStore/{type}
   ```

### 查看圖片列表

頁面會顯示：

| 列名 | 說明 |
|------|------|
| **預覽** | 圖片縮略圖 |
| **名稱** | 圖片文件名 |
| **描述** | 圖片描述 |
| **上傳日期** | 上傳時間 |
| **尺寸** | 圖片尺寸（像素） |
| **大小** | 文件大小（KB） |
| **操作** | 查看、刪除 |

### 上傳圖片

#### 步驟 1：選擇圖片類型

在應用配置頁面選擇要上傳的圖片類型：
- banner
- featured
- brand
- campaign

#### 步驟 2：上傳圖片

1. 點擊**"上傳圖片"**或**"Upload"**
2. 選擇圖片文件
3. 輸入圖片描述（可選）
   ```
   URL: /cms/application/imageStore/{type}/upload (POST)
   ```
4. 點擊**"上傳"**

#### 支持的圖片格式

- ✅ JPEG (.jpg)
- ✅ PNG (.png)
- ✅ GIF (.gif)
- ❌ 不支持 BMP、TIFF 等格式

#### 圖片要求

**尺寸要求**：
- 輪播圖：建議 1080x1920 像素（寬x高）
- 特色項目：建議 1080x1080 像素（正方形）
- 品牌圖片：建議 512x512 像素

**文件大小**：
- 最大 5MB
- 建議使用壓縮後的圖片

**質量要求**：
- 分辨率 72 DPI（網頁顯示）
- 避免使用模糊或低質量圖片

### 圖片管理操作

#### 查看圖片詳情

點擊圖片預覽可以查看：
- 原始尺寸圖片
- 完整的描述信息
- 上傳時間和文件信息

#### 刪除圖片

1. 找到要刪除的圖片
2. 點擊**"刪除"**按鈕
3. 確認刪除

**注意事項**：
- ⚠️ 刪除圖片會影響正在使用該圖片的地方
- ⚠️ 刪除前確認沒有頁面配置引用該圖片
- ⚠️ 刪除操作不可逆

## 🏷️ 標簽管理（Tag Groups）

### 什麼是標簽組

標簽組用於分類和組織內容，包括：

| 標簽組類型 | 用途 |
|-----------|------|
| **品牌標簽** | 按品牌分組內容 |
| **地理標簽** | 按地理位置分組內容 |

### 訪問標簽管理

1. 在應用配置頁面
2. 點擊**"標簽管理"**或**"Tag Group"**
3. 系統會顯示所有標簽組
   ```
   URL: /cms/application/tagGroup
   ```

### 查看標簽組列表

顯示所有標簽組：
- 標簽組 ID
- 標簽組名稱
- 標簽組類型（品牌/地理）
- 標簽數量
- 操作（編輯、刪除）

### 創建標簽組

#### 步驟 1：點擊新建

1. 在標簽組列表頁面
2. 點擊**"新建標簽組"**

#### 步驟 2：填寫標簽組信息

**基本信息**：
- **標簽組名稱**：標簽組的顯示名稱
- **標簽組類型**：
  - `brand`：品牌標簽組
  - `geo`：地理標簽組

**標簽配置**：
- **標簽列表**：添加標簽
- **標簽順序**：拖動調整順序
- **標簽顏色**：設置標簽顏色（可選）

#### 步驟 3：保存標簽組

1. 點擊**"保存"**
   ```
   URL: /cms/application/tagGroup/{id} (POST)
   ```
2. 系統會創建標簽組
3. 可以在前端頁面設置中使用

### 編輯標簽組

1. 在標簽組列表中找到要編輯的標簽組
2. 點擊標簽組名稱進入編輯頁面
   ```
   URL: /cms/application/tagGroup/{id} (GET)
   ```
3. 修改標簽組信息
4. 點擊**"保存"**

### 刪除標簽組

1. 在標簽組列表中找到要刪除的標簽組
2. 點擊**"刪除"**
3. 確認刪除

**注意事項**：
- ⚠️ 刪除標簽組會影響使用該標簽組的配置
- ⚠️ 刪除操作不可逆

## 🌍 地區管理（Region）

### 什麼是地區管理

地區管理用於配置應用支持的國家和地區，包括：
- 國家名稱
- 國家代碼
- 貨幣
- 語言

### 訪問地區管理

1. 在應用配置頁面
2. 點擊**"地區管理"**或**"Region"**
3. 系統會顯示所有地區
   ```
   URL: /cms/application/region
   ```

### 查看地區列表

顯示所有配置的地區：

| 字段 | 說明 |
|------|------|
| **地區名稱** | 國家/地區名稱 |
| **國家代碼** | ISO 國家代碼（如 HK, CN, US） |
| **貨幣** | 貨幣代碼（如 HKD, CNY, USD） |
| **語言** | 默認語言 |
| **狀態** | 啟用/禁用 |

### 創建地區

#### 步驟 1：點擊新建

1. 在地區列表頁面
2. 點擊**"新建地區"**

#### 步驟 2：填寫地區信息

**基本信息**：
- **地區名稱**：國家/地區的顯示名稱
  - 例如：香港、中國、美國
- **國家代碼**：ISO 3166-1 alpha-2 代碼
  - 例如：HK、CN、US
- **貨幣**：ISO 4217 貨幣代碼
  - 例如：HKD、CNY、USD
- **默認語言**：選擇默認語言
  - 例如：zh-HK（繁體中文香港）、zh-CN（簡體中文）

**高級設置**：
- **時區**：選擇時區
- **日期格式**：選擇日期顯示格式
- **數字格式**：選擇數字顯示格式

#### 步驟 3：保存地區

1. 點擊**"保存"**
   ```
   URL: /cms/application/region/{id} (POST)
   ```
2. 系統會創建地區配置
3. 用戶可以選擇該地區顯示相應內容

### 編輯地區

1. 在地區列表中找到要編輯的地區
2. 點擊地區名稱進入編輯頁面
   ```
   URL: /cms/application/region/{id} (GET)
   ```
3. 修改地區信息
4. 點擊**"保存"**

### 刪除地區

1. 在地區列表中找到要刪除的地區
2. 點擊**"刪除"**
3. 確認刪除

**注意事項**：
- ⚠️ 刪除地區會影響該地區的用戶
- ⚠️ 刪除前確保沒有用戶使用該地區設置
- ⚠️ 刪除操作不可逆

## 🔑 POS 密鑰管理（POS Keys）

### 什麼是 POS 密鑰

POS 密鑰用於 POS 系統與 CMS 系統之間的安全通信。每個 POS 終端有唯一的密鑰對。

### 訪問 POS 密鑰管理

1. 在應用配置頁面
2. 點擊**"POS 密鑰"**或**"POS Keys"**
3. 系統會顯示所有 POS 密鑰
   ```
   URL: /cms/application/viewposkeys
   ```

### 查看所有 POS 密鑰

顯示所有 POS 終端的密鑰信息：

| 字段 | 說明 |
|------|------|
| **POS ID** | POS 終端唯一標識 |
| **店鋪** | 所屬店鋪 |
| **狀態** | 活躍/測試/已停用 |
| **創建日期** | 密鑰創建時間 |
| **有效期** | 密鑰過期時間 |
| **操作** | 查看、輪換、提升、回滾 |

### 查看單個 POS 密鑰

1. 在 POS 密鑰列表中
2. 點擊 POS ID 查看詳情
   ```
   URL: /cms/application/viewposkey/{posId}
   URL: /cms/application/poskey/{posId} (GET)
   ```

### 創建 POS 密鑰

#### 步驟 1：點擊新建

1. 在 POS 密鑰列表頁面
2. 點擊**"新建 POS 密鑰"**

#### 步驟 2：填寫 POS 信息

- **POS ID**：輸入 POS 終端唯一標識
- **店鋪**：選擇所屬店鋪
- **密鑰類型**：
  - Production：生產環境密鑰
  - Test：測試環境密鑰

#### 步驟 3：生成密鑰

1. 點擊**"生成密鑰"**
2. 系統會生成：
   - 密鑰（Key）
   - 密碼（Secret）
3. **重要**：安全保存密鑰信息

#### 步驟 4：保存密鑰

1. 點擊**"保存"**
   ```
   URL: /cms/application/poskey/{posId} (POST)
   ```
2. 系統會創建 POS 密鑰

### 輪換 POS 密鑰（Rotate）

**場景**：定期更新密鑰提高安全性

1. 找到要輪換的 POS 密鑰
2. 點擊**"輪換密鑰"**或**"Rotate"**
   ```
   URL: /cms/application/poskey/{posId}/rotate (POST)
   ```
3. 系統會：
   - 生成新的密鑰對
   - 舊密鑰標記為"即將過期"
   - 新密鑰立即可用

**輪換過程**：
- ✅ 舊密鑰繼續工作一段時間（grace period）
- ✅ 新密鑰可以立即使用
- ✅ 需要在 POS 終端更新密鑰

### 提升密鑰（Promote）

**場景**：將測試密鑰提升為生產密鑰

1. 找到要提升的測試密鑰
2. 點擊**"提升為生產密鑰"**或**"Promote"**
   ```
   URL: /cms/application/poskey/{posId}/promote (POST)
   ```
3. 系統會將密鑰類型從 Test 改為 Production

**注意事項**：
- ⚠️ 提升後密鑰將用於生產環境
- ⚠️ 確保密鑰配置正確
- ⚠️ 無法從生產降回測試

### 回滾密鑰（Rollback）

**場景**：回退到上一個版本的密鑰

1. 找到要回滾的 POS 密鑰
2. 點擊**"回滾"**或**"Rollback"**
   ```
   URL: /cms/application/poskey/{posId}/rollback (POST)
   ```
3. 系統會恢復到上一個密鑰版本

**注意事項**：
- ⚠️ 回滾會使當前密鑰失效
- ⚠️ 需要在 POS 終端恢復舊密鑰
- ⚠️ 僅限回滾一次

### 停用舊密鑰（Retire Previous）

**場景**：在輪換密鑰後停用舊密鑰

1. 在輪換密鑰後
2. 等待足夠的時間（確保所有終端已更新）
3. 點擊**"停用舊密鑰"**或**"Retire Previous"**
   ```
   URL: /cms/application/poskey/{posId}/retirePrevious (POST)
   ```
4. 舊密鑰將完全失效

### 測試 POS 密鑰

1. 找到要測試的 POS 密鑰
2. 點擊**"測試"**或**"Test"**
   ```
   URL: /cms/application/testposkey/{posId} (GET)
   ```
3. 系統會顯示：
   - 密鑰是否有效
   - 密鑰過期時間
   - 連接狀態

## 🔄 PMS 集成記錄（PMS Records）

### 什麼是 PMS 集成

PMS（Property Management System）集成用於與酒店管理系統對接，實現：
- 會員信息同步
- 交易數據同步
- 積分兌換

### 訪問 PMS 記錄

1. 在應用配置頁面
2. 點擊**"PMS 記錄"**或**"PMS Records"**
3. 系統會顯示所有 PMS 集成記錄
   ```
   URL: /cms/application/pmsRecordsView
   ```

### 查看所有 PMS 記錄

顯示所有 PMS 集成記錄：

| 字段 | 說明 |
|------|------|
| **記錄 ID** | 唯一標識 |
| **品牌 ID** | 所屬品牌 |
| **店鋪參考 ID** | 店鋪在 PMS 系統中的 ID |
| **狀態** | 同步狀態 |
| **最後同步** | 最後同步時間 |

### 篩選 PMS 記錄

**按品牌篩選**：
- 選擇特定品牌
- 查看該品牌的 PMS 記錄
   ```
   URL: /cms/application/pmsRecords/{brandId}
   ```

### 查看單個 PMS 記錄

1. 在 PMS 記錄列表中
2. 點擊記錄 ID 查看詳情
   ```
   URL: /cms/application/pmsRecordView/{id}
   URL: /cms/application/pmsRecord/{id} (GET)
   URL: /cms/application/pmsRecord/{brandId}/{shopReferenceId} (GET)
   ```

### 創建 PMS 記錄

#### 步驟 1：點擊新建

1. 在 PMS 記錄列表頁面
2. 點擊**"新建 PMS 記錄"**

#### 步驟 2：填寫 PMS 信息

**基本信息**：
- **品牌 ID**：選擇品牌
- **店鋪參考 ID**：輸入店鋪在 PMS 中的 ID
- **配置信息**：輸入 PMS 連接配置

#### 步驟 3：保存 PMS 記錄

1. 點擊**"保存"**
   ```
   URL: /cms/application/pmsRecord/{brandId}/{shopReferenceId} (POST)
   URL: /cms/application/pmsRecord/{id} (POST)
   ```
2. 系統會創建 PMS 集成記錄

### 編輯 PMS 記錄

1. 找到要編輯的 PMS 記錄
2. 進入記錄詳情頁面
3. 修改配置信息
4. 點擊**"保存"**

### 刪除 PMS 記錄

1. 找到要刪除的 PMS 記錄
2. 點擊**"刪除"**
   ```
   URL: /cms/application/pmsRecord/{id} (DELETE)
   URL: /cms/application/pmsRecord/{brandId}/{shopReferenceId} (DELETE)
   ```
3. 確認刪除

**注意事項**：
- ⚠️ 刪除 PMS 記錄會停止同步
- ⚠️ 刪除操作不可逆

## 🔧 常見問題

### Q: 前端頁面配置不生效？

**A**：
1. 確認已保存配置
2. 清除應用緩存重新加載
3. 檢查 JSON 格式是否正確
4. 確認配置中引用的圖片 ID 存在

### Q: 圖片上傳失敗？

**A**：
1. 檢查圖片格式是否支持（JPEG/PNG/GIF）
2. 確認圖片大小不超過 5MB
3. 嘗試壓縮圖片後重新上傳
4. 檢查網絡連接

### Q: POS 密鑰輪換後終端無法連接？

**A**：
1. 確認 POS 終端已更新密鑰
2. 檢查是否在 grace period 內
3. 如果已過 grace period，需要回滾密鑰
4. 聯系技術支持協助

### Q: 如何備份當前配置？

**A**：
1. 前端頁面設置：導出 JSON 配置保存
2. 標簽組：記錄標簽組配置
3. POS 密鑰：安全保存密鑰文件
4. 應用配置：導出配置列表
