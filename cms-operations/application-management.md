# CMS 应用配置管理指南

## 标签
#user-guide #application #settings #advanced

## 概述

本文档介绍如何配置和管理 Storellet CMS 应用设置，包括前端页面配置、图片管理、标签管理、地区设置、POS 密钥管理和 PMS 集成。

---

## 🔧 访问应用配置

### 步骤 1：进入应用配置页面

1. 登录 CMS 系统
2. 点击顶部导航的**"应用"**或**"Application"**
3. 系统会跳转到应用配置首页
   ```
   URL: /cms/application/index
   ```

---

## 🎨 前端页面设置（Front Page Settings）

### 什么是前端页面设置

前端页面设置用于配置应用首页的布局和显示内容，包括：
- 轮播图
- 特色项目
- 品牌分组
- 地理位置分组

### 访问前端页面设置

1. 在应用配置页面
2. 点击**"前端页面设置"**或**"Front Page Setting"**
3. 系统会显示当前的前端页面配置
   ```
   URL: /cms/application/frontPageSetting
   ```

### 查看当前配置

页面会显示：

#### 1. 当前配置 JSON

显示当前的页面布局配置：
```json
[
  {
    "type": "banner",
    "items": [...]
  },
  {
    "type": "featured",
    "items": [...]
  }
]
```

#### 2. 品牌标签组（Tag Groups）

显示可用的品牌标签：
- 标签组名称
- 标签列表
- 标签顺序

#### 3. 地理标签组（Geo Tag Groups）

显示可用的地理标签：
- 地理区域名称
- 区域内的标签

#### 4. 特色项目（Featured Items）

显示当前的特色项目列表：
- 活动名称
- 图片
- 链接

### 修改前端页面配置

#### 步骤 1：编辑 JSON 配置

1. 在 JSON 编辑器中修改配置
2. 配置类型包括：
   - **banner**：轮播图配置
   - **featured**：特色项目配置
   - **tagGroup**：标签分组配置

#### 步骤 2：配置轮播图

```json
{
  "type": "banner",
  "items": [
    {
      "imageId": "image123",
      "link": "https://example.com",
      "title": "Banner 1"
    }
  ]
}
```

#### 步骤 3：配置特色项目

```json
{
  "type": "featured",
  "items": [
    {
      "campaignName": "活动名称",
      "imageUrl": "https://example.com/image.jpg",
      "link": "campaign://123"
    }
  ]
}
```

#### 步骤 4：保存配置

1. 点击**"保存"**或**"Update"**
   ```
   URL: /cms/application/frontPageSetting (POST)
   ```
2. 系统会：
   - 创建新的配置记录
   - 将旧配置设为非活跃状态
   - 新配置立即生效

### 配置最佳实践

#### 轮播图配置
- ✅ 使用高质量的图片（建议 1080x1920 像素）
- ✅ 每张图片添加描述性标题
- ✅ 设置正确的跳转链接
- ❌ 不要添加过多轮播图（建议 3-5 张）

#### 特色项目配置
- ✅ 选择当前热门的活动
- ✅ 定期更新内容
- ✅ 使用吸引人的图片
- ❌ 不要显示已结束的活动

## 🖼️ 图片管理（Image Store）

### 什么是图片存储

图片存储用于集中管理应用中使用的所有图片，按类型分类：
- **banner**：轮播图
- **featured**：特色项目图片
- **brand**：品牌图片
- **campaign**：活动图片

### 访问图片管理

1. 在应用配置页面
2. 选择要管理的图片类型
3. 系统会显示该类型的所有图片
   ```
   URL: /cms/application/imageStore/{type}
   ```

### 查看图片列表

页面会显示：

| 列名 | 说明 |
|------|------|
| **预览** | 图片缩略图 |
| **名称** | 图片文件名 |
| **描述** | 图片描述 |
| **上传日期** | 上传时间 |
| **尺寸** | 图片尺寸（像素） |
| **大小** | 文件大小（KB） |
| **操作** | 查看、删除 |

### 上传图片

#### 步骤 1：选择图片类型

在应用配置页面选择要上传的图片类型：
- banner
- featured
- brand
- campaign

#### 步骤 2：上传图片

1. 点击**"上传图片"**或**"Upload"**
2. 选择图片文件
3. 输入图片描述（可选）
   ```
   URL: /cms/application/imageStore/{type}/upload (POST)
   ```
4. 点击**"上传"**

#### 支持的图片格式

- ✅ JPEG (.jpg)
- ✅ PNG (.png)
- ✅ GIF (.gif)
- ❌ 不支持 BMP、TIFF 等格式

#### 图片要求

**尺寸要求**：
- 轮播图：建议 1080x1920 像素（宽x高）
- 特色项目：建议 1080x1080 像素（正方形）
- 品牌图片：建议 512x512 像素

**文件大小**：
- 最大 5MB
- 建议使用压缩后的图片

**质量要求**：
- 分辨率 72 DPI（网页显示）
- 避免使用模糊或低质量图片

### 图片管理操作

#### 查看图片详情

点击图片预览可以查看：
- 原始尺寸图片
- 完整的描述信息
- 上传时间和文件信息

#### 删除图片

1. 找到要删除的图片
2. 点击**"删除"**按钮
3. 确认删除

**注意事项**：
- ⚠️ 删除图片会影响正在使用该图片的地方
- ⚠️ 删除前确认没有页面配置引用该图片
- ⚠️ 删除操作不可逆

## 🌍 地区管理（Region）

### 什么是地区管理

地区管理用于配置应用支持的国家和地区，包括：
- 国家名称
- 国家代码
- 货币
- 语言

### 访问地区管理

1. 在应用配置页面
2. 点击**"地区管理"**或**"Region"**
3. 系统会显示所有地区
   ```
   URL: /cms/application/region
   ```

### 查看地区列表

显示所有配置的地区：

| 字段 | 说明 |
|------|------|
| **地区名称** | 国家/地区名称 |
| **国家代码** | ISO 国家代码（如 HK, CN, US） |
| **货币** | 货币代码（如 HKD, CNY, USD） |
| **语言** | 默认语言 |
| **状态** | 启用/禁用 |

### 创建地区

#### 步骤 1：点击新建

1. 在地区列表页面
2. 点击**"新建地区"**

#### 步骤 2：填写地区信息

**基本信息**：
- **地区名称**：国家/地区的显示名称
  - 例如：香港、中国、美国
- **国家代码**：ISO 3166-1 alpha-2 代码
  - 例如：HK、CN、US
- **货币**：ISO 4217 货币代码
  - 例如：HKD、CNY、USD
- **默认语言**：选择默认语言
  - 例如：zh-HK（繁体中文香港）、zh-CN（简体中文）

**高级设置**：
- **时区**：选择时区
- **日期格式**：选择日期显示格式
- **数字格式**：选择数字显示格式

#### 步骤 3：保存地区

1. 点击**"保存"**
   ```
   URL: /cms/application/region/{id} (POST)
   ```
2. 系统会创建地区配置
3. 用户可以选择该地区显示相应内容

### 编辑地区

1. 在地区列表中找到要编辑的地区
2. 点击地区名称进入编辑页面
   ```
   URL: /cms/application/region/{id} (GET)
   ```
3. 修改地区信息
4. 点击**"保存"**

### 删除地区

1. 在地区列表中找到要删除的地区
2. 点击**"删除"**
3. 确认删除

**注意事项**：
- ⚠️ 删除地区会影响该地区的用户
- ⚠️ 删除前确保没有用户使用该地区设置
- ⚠️ 删除操作不可逆

## 🔑 POS 密钥管理（POS Keys）

### 什么是 POS 密钥

POS 密钥用于 POS 系统与 CMS 系统之间的安全通信。每个 POS 终端有唯一的密钥对。

### 访问 POS 密钥管理

1. 在应用配置页面
2. 点击**"POS 密钥"**或**"POS Keys"**
3. 系统会显示所有 POS 密钥
   ```
   URL: /cms/application/viewposkeys
   ```

### 查看所有 POS 密钥

显示所有 POS 终端的密钥信息：

| 字段 | 说明 |
|------|------|
| **POS ID** | POS 终端唯一标识 |
| **店铺** | 所属店铺 |
| **状态** | 活跃/测试/已停用 |
| **创建日期** | 密钥创建时间 |
| **有效期** | 密钥过期时间 |
| **操作** | 查看、轮换、提升、回滚 |

### 查看单个 POS 密钥

1. 在 POS 密钥列表中
2. 点击 POS ID 查看详情
   ```
   URL: /cms/application/viewposkey/{posId}
   URL: /cms/application/poskey/{posId} (GET)
   ```

### 创建 POS 密钥

#### 步骤 1：点击新建

1. 在 POS 密钥列表页面
2. 点击**"新建 POS 密钥"**

#### 步骤 2：填写 POS 信息

- **POS ID**：输入 POS 终端唯一标识
- **店铺**：选择所属店铺
- **密钥类型**：
  - Production：生产环境密钥
  - Test：测试环境密钥

#### 步骤 3：生成密钥

1. 点击**"生成密钥"**
2. 系统会生成：
   - 密钥（Key）
   - 密码（Secret）
3. **重要**：安全保存密钥信息

#### 步骤 4：保存密钥

1. 点击**"保存"**
   ```
   URL: /cms/application/poskey/{posId} (POST)
   ```
2. 系统会创建 POS 密钥

### 轮换 POS 密钥（Rotate）

**场景**：定期更新密钥提高安全性

1. 找到要轮换的 POS 密钥
2. 点击**"轮换密钥"**或**"Rotate"**
   ```
   URL: /cms/application/poskey/{posId}/rotate (POST)
   ```
3. 系统会：
   - 生成新的密钥对
   - 旧密钥标记为"即将过期"
   - 新密钥立即可用

**轮换过程**：
- ✅ 旧密钥继续工作一段时间（grace period）
- ✅ 新密钥可以立即使用
- ✅ 需要在 POS 终端更新密钥

### 提升密钥（Promote）

**场景**：将测试密钥提升为生产密钥

1. 找到要提升的测试密钥
2. 点击**"提升为生产密钥"**或**"Promote"**
   ```
   URL: /cms/application/poskey/{posId}/promote (POST)
   ```
3. 系统会将密钥类型从 Test 改为 Production

**注意事项**：
- ⚠️ 提升后密钥将用于生产环境
- ⚠️ 确保密钥配置正确
- ⚠️ 无法从生产降回测试

### 回滚密钥（Rollback）

**场景**：回退到上一个版本的密钥

1. 找到要回滚的 POS 密钥
2. 点击**"回滚"**或**"Rollback"**
   ```
   URL: /cms/application/poskey/{posId}/rollback (POST)
   ```
3. 系统会恢复到上一个密钥版本

**注意事项**：
- ⚠️ 回滚会使当前密钥失效
- ⚠️ 需要在 POS 终端恢复旧密钥
- ⚠️ 仅限回滚一次

### 停用旧密钥（Retire Previous）

**场景**：在轮换密钥后停用旧密钥

1. 在轮换密钥后
2. 等待足够的时间（确保所有终端已更新）
3. 点击**"停用旧密钥"**或**"Retire Previous"**
   ```
   URL: /cms/application/poskey/{posId}/retirePrevious (POST)
   ```
4. 旧密钥将完全失效

### 测试 POS 密钥

1. 找到要测试的 POS 密钥
2. 点击**"测试"**或**"Test"**
   ```
   URL: /cms/application/testposkey/{posId} (GET)
   ```
3. 系统会显示：
   - 密钥是否有效
   - 密钥过期时间
   - 连接状态

## 🔄 PMS 集成记录（PMS Records）

### 什么是 PMS 集成

PMS（Property Management System）集成用于与酒店管理系统对接，实现：
- 会员信息同步
- 交易数据同步
- 积分兑换

### 访问 PMS 记录

1. 在应用配置页面
2. 点击**"PMS 记录"**或**"PMS Records"**
3. 系统会显示所有 PMS 集成记录
   ```
   URL: /cms/application/pmsRecordsView
   ```

### 查看所有 PMS 记录

显示所有 PMS 集成记录：

| 字段 | 说明 |
|------|------|
| **记录 ID** | 唯一标识 |
| **品牌 ID** | 所属品牌 |
| **店铺参考 ID** | 店铺在 PMS 系统中的 ID |
| **状态** | 同步状态 |
| **最后同步** | 最后同步时间 |

### 筛选 PMS 记录

**按品牌筛选**：
- 选择特定品牌
- 查看该品牌的 PMS 记录
   ```
   URL: /cms/application/pmsRecords/{brandId}
   ```

### 查看单个 PMS 记录

1. 在 PMS 记录列表中
2. 点击记录 ID 查看详情
   ```
   URL: /cms/application/pmsRecordView/{id}
   URL: /cms/application/pmsRecord/{id} (GET)
   URL: /cms/application/pmsRecord/{brandId}/{shopReferenceId} (GET)
   ```

### 创建 PMS 记录

#### 步骤 1：点击新建

1. 在 PMS 记录列表页面
2. 点击**"新建 PMS 记录"**

#### 步骤 2：填写 PMS 信息

**基本信息**：
- **品牌 ID**：选择品牌
- **店铺参考 ID**：输入店铺在 PMS 中的 ID
- **配置信息**：输入 PMS 连接配置

#### 步骤 3：保存 PMS 记录

1. 点击**"保存"**
   ```
   URL: /cms/application/pmsRecord/{brandId}/{shopReferenceId} (POST)

   ```

### 编辑 PMS 记录

1. 找到要编辑的 PMS 记录
2. 进入记录详情页面
3. 修改配置信息
4. 点击**"保存"**

### 删除 PMS 记录

1. 找到要删除的 PMS 记录
2. 点击**"删除"**
   ```
   URL: /cms/application/pmsRecord/{id} (DELETE)
   URL: /cms/application/pmsRecord/{brandId}/{shopReferenceId} (DELETE)
   ```
3. 确认删除

**注意事项**：
- ⚠️ 删除 PMS 记录会停止同步
- ⚠️ 删除操作不可逆

## 🔧 常见问题

### Q: 前端页面配置不生效？

**A**：
1. 确认已保存配置
2. 清除应用缓存重新加载
3. 检查 JSON 格式是否正确
4. 确认配置中引用的图片 ID 存在

### Q: 图片上传失败？

**A**：
1. 检查图片格式是否支持（JPEG/PNG/GIF）
2. 确认图片大小不超过 5MB
3. 尝试压缩图片后重新上传
4. 检查网络连接

### Q: POS 密钥轮换后终端无法连接？

**A**：
1. 确认 POS 终端已更新密钥
2. 检查是否在 grace period 内
3. 如果已过 grace period，需要回滚密钥
4. 联系技术支持协助

### Q: 如何备份当前配置？

**A**：
1. 前端页面设置：导出 JSON 配置保存
2. 标签组：记录标签组配置
3. POS 密钥：安全保存密钥文件
4. 应用配置：导出配置列表

## 🎯 最佳实践

### 配置管理

#### 版本控制
- ✅ 重要配置修改前先备份
- ✅ 记录配置修改历史
- ✅ 定期导出配置保存

#### 权限管理
- ✅ 配置修改需要授权
- ✅ 敏感配置（POS 密钥）限制访问
- ✅ 记录所有配置修改日志

### POS 密钥管理

#### 定期轮换
- ✅ 每 3-6 个月轮换一次 POS 密钥
- ✅ 轮换后及时更新所有终端
- ✅ 旧密钥保留一段时间后停用

#### 密钥安全
- ✅ 不要通过邮件发送密钥
- ✅ 使用加密方式传输密钥
- ✅ 定期审计密钥使用记录

### 前端页面配置

#### 内容更新
- ✅ 定期更新轮播图内容
- ✅ 及时移除已结束的活动
- ✅ 优化图片加载速度

#### 用户体验
- ✅ 确保配置的链接有效
- ✅ 图片质量优化
- ✅ 移动端适配检查
