# SEO 優化指南

## 標簽
#operation #seo #beginner

## 概述

本文檔介紹如何在 Storellet CMS 中優化內容的 SEO（搜索引擎優化）。

## SEO 檢查項目

### 必檢項目

#### 1. 標題優化

**要求**：
- 標題長度：50-60 個字符
- 包含主要關鍵詞
- 吸引人點擊

**示例**：
```
好的標題：如何創建和管理 CMS 用戶賬戶 | 完整指南
不好的標題：用戶管理
```

#### 2. Meta 描述

**要求**：
- 長度：150-160 個字符
- 包含關鍵詞
- 概括頁面內容

**示例**：
```
學習如何在 Storellet CMS 中創建、編輯和管理用戶賬戶。
完整的步驟指南，包括批量導入、權限設置和常見問題解答。
```

#### 3. 關鍵詞密度

**最佳實踐**：
- 主要關鍵詞密度：1-2%
- 避免關鍵詞堆砌
- 使用同義詞和相關詞

#### 4. 內容長度

**建議**：
- 最少：300 字
- 推薦：1000-1500 字
- 最佳：2000+ 字（深度內容）

### 可選項目

#### 5. URL 結構

**好的 URL**：
```
✅ /cms-user-management-guide
❌ /post?id=123
```

#### 6. 標題層級

```markdown
# H1: 主標題（每頁只有一個）
## H2: 主要章節
### H3: 子章節
#### H4: 細節
```

#### 7. 圖片優化

- 使用描述性的文件名：`user-management-flow.png`
- 添加 alt 文本：`<img src="..." alt="用戶管理流程圖">`
- 壓縮圖片大小

## 在 CMS 中設置 SEO

### 編輯內容時

1. **填寫 SEO 字段**：
   - SEO 標題
   - Meta 描述
   - 關鍵詞

2. **使用 AI 輔助**：
   - 點擊 "AI 優化" 按鈕
   - 選擇 "SEO 優化"
   - AI 會建議改進

3. **檢查 SEO 分數**：
   - CMS 會顯示 SEO 分數（0-100）
   - 目標：80 分以上

### 發布前檢查清單

- [ ] 標題長度 50-60 字符
- [ ] Meta 描述 150-160 字符
- [ ] 關鍵詞密度 1-2%
- [ ] 內容長度 > 300 字
- [ ] 包含 H1-H6 標題
- [ ] 圖片有 alt 文本
- [ ] URL 簡潔明了
- [ ] SEO 分數 > 80

## SEO 分數計算

CMS SEO 檢查會評估以下項目：

| 項目 | 權重 | 說明 |
|------|------|------|
| 標題 | 20% | 長度、關鍵詞 |
| 描述 | 20% | 長度、吸引力 |
| 關鍵詞 | 15% | 密度、分布 |
| 內容 | 20% | 長度、質量 |
| 結構 | 15% | 標題層級 |
| 多媒體 | 10% | 圖片、視頻 |

**分數計算**：
```
SEO 分數 = 各項加權平均
目標分數：80/100 以上
```

## 常見問題

### Q: SEO 分數低怎麼辦？

**A**：
1. 查看 AI 建議的具體改進點
2. 優先處理權重高的項目（標題、描述、內容）
3. 使用 AI 自動優化功能

### Q: 關鍵詞密度多少合適？

**A**：
- 主要關鍵詞：1-2%
- 長尾關鍵詞：0.5-1%
- 自然寫作，不要堆砌

### Q: 如何提高搜索排名？

**A**：
1. 持續發布高質量內容
2. 優化現有內容的 SEO
3. 建立內部鏈接
4. 獲得外部鏈接
5. 保持內容更新

## 最佳實踐

### 內容創作

1. **關鍵詞研究**：
   - 使用 Google Keyword Planner
   - 分析競爭對手
   - 找到長尾關鍵詞

2. **內容結構**：
   - 使用清晰的標題層級
   - 添加目錄（可選）
   - 使用要點列表

3. **內容更新**：
   - 定期更新舊內容
   - 添加最新信息
   - 保持內容准確性

### 技術優化

1. **頁面速度**：
   - 壓縮圖片
   - 減少 HTTP 請求
   - 使用 CDN

2. **移動友好**：
   - 響應式設計
   - 移動速度測試
   - 移動可用性

3. **結構化數據**：
   - 添加 Schema 標記
   - 使用面包屑
   - 標記作者信息
