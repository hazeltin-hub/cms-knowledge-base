# SEO 优化指南

## 标签
#operation #seo #beginner

## 概述

本文档介绍如何在 Storellet CMS 中优化内容的 SEO（搜索引擎优化）。

## SEO 检查项目

### 必检项目

#### 1. 标题优化

**要求**：
- 标题长度：50-60 个字符
- 包含主要关键词
- 吸引人点击

**示例**：
```
好的标题：如何创建和管理 CMS 用户账户 | 完整指南
不好的标题：用户管理
```

#### 2. Meta 描述

**要求**：
- 长度：150-160 个字符
- 包含关键词
- 概括页面内容

**示例**：
```
学习如何在 Storellet CMS 中创建、编辑和管理用户账户。
完整的步骤指南，包括批量导入、权限设置和常见问题解答。
```

#### 3. 关键词密度

**最佳实践**：
- 主要关键词密度：1-2%
- 避免关键词堆砌
- 使用同义词和相关词

#### 4. 内容长度

**建议**：
- 最少：300 字
- 推荐：1000-1500 字
- 最佳：2000+ 字（深度内容）

### 可选项目

#### 5. URL 结构

**好的 URL**：
```
✅ /cms-user-management-guide
❌ /post?id=123
```

#### 6. 标题层级

```markdown
# H1: 主标题（每页只有一个）
## H2: 主要章节
### H3: 子章节
#### H4: 细节
```

#### 7. 图片优化

- 使用描述性的文件名：`user-management-flow.png`
- 添加 alt 文本：`<img src="..." alt="用户管理流程图">`
- 压缩图片大小

## 在 CMS 中设置 SEO

### 编辑内容时

1. **填写 SEO 字段**：
   - SEO 标题
   - Meta 描述
   - 关键词

2. **使用 AI 辅助**：
   - 点击 "AI 优化" 按钮
   - 选择 "SEO 优化"
   - AI 会建议改进

3. **检查 SEO 分数**：
   - CMS 会显示 SEO 分数（0-100）
   - 目标：80 分以上

### 发布前检查清单

- [ ] 标题长度 50-60 字符
- [ ] Meta 描述 150-160 字符
- [ ] 关键词密度 1-2%
- [ ] 内容长度 > 300 字
- [ ] 包含 H1-H6 标题
- [ ] 图片有 alt 文本
- [ ] URL 简洁明了
- [ ] SEO 分数 > 80

## SEO 分数计算

CMS SEO 检查会评估以下项目：

| 项目 | 权重 | 说明 |
|------|------|------|
| 标题 | 20% | 长度、关键词 |
| 描述 | 20% | 长度、吸引力 |
| 关键词 | 15% | 密度、分布 |
| 内容 | 20% | 长度、质量 |
| 结构 | 15% | 标题层级 |
| 多媒体 | 10% | 图片、视频 |

**分数计算**：
```
SEO 分数 = 各项加权平均
目标分数：80/100 以上
```

## 常见问题

### Q: SEO 分数低怎么办？

**A**：
1. 查看 AI 建议的具体改进点
2. 优先处理权重高的项目（标题、描述、内容）
3. 使用 AI 自动优化功能

### Q: 关键词密度多少合适？

**A**：
- 主要关键词：1-2%
- 长尾关键词：0.5-1%
- 自然写作，不要堆砌

### Q: 如何提高搜索排名？

**A**：
1. 持续发布高质量内容
2. 优化现有内容的 SEO
3. 建立内部链接
4. 获得外部链接
5. 保持内容更新

## 最佳实践

### 内容创作

1. **关键词研究**：
   - 使用 Google Keyword Planner
   - 分析竞争对手
   - 找到长尾关键词

2. **内容结构**：
   - 使用清晰的标题层级
   - 添加目录（可选）
   - 使用要点列表

3. **内容更新**：
   - 定期更新旧内容
   - 添加最新信息
   - 保持内容准确性

### 技术优化

1. **页面速度**：
   - 压缩图片
   - 减少 HTTP 请求
   - 使用 CDN

2. **移动友好**：
   - 响应式设计
   - 移动速度测试
   - 移动可用性

3. **结构化数据**：
   - 添加 Schema 标记
   - 使用面包屑
   - 标记作者信息
